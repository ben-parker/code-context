## Revised Plan: Implementing TypeScript Parsing via Node.js Bridge

**Objective:** Implement a TypeScript/JavaScript parser by creating a bridge from the C# application to a dedicated Node.js script. This script will leverage the official `typescript` NPM package to generate an AST, transform it into our specified `CodeGraph` JSON format, and return it to the C# process via standard output.

This approach will serve as the template for "out-of-process" language parsers.

### Core Concept: The Process Bridge

The C# `TypeScriptParser` will not parse code directly. Its sole responsibility will be to:
1.  Invoke a Node.js script using `npx` (or `node`).
2.  Pass the source code content to the script via `stdin`.
3.  Receive a `CodeGraph` JSON string from the script's `stdout`.
4.  Deserialize the JSON into the C# `CodeGraph` object.

```
┌───────────────────────────┐      ┌────────────────────────────────┐      ┌──────────────────────────┐
│ C# Process                │      │       Standard I/O Bridge        │      │ Node.js Process          │
│                           │      │                                │      │                          │
│ TypeScriptParser.ParseFile│───┐  │                                │  ┌───▶│ parser.js (stdin)      │
│ (file content)            │   └─▶│ process.StandardInput.Write()  │──┘     │ (uses 'typescript' npm)  │
│                           │      │                                │      │                          │
│                           │      │                                │      │ (transforms AST to JSON) │
│                           │   ┌──│ process.StandardOutput.Read()  │◀─┐     │                          │
│ CodeGraph (C# Object)     │◀──┘  │                                │  └────│ console.log(json) (stdout)│
└───────────────────────────┘      └────────────────────────────────┘      └──────────────────────────┘
```

### 2. The Node.js Parser Script

This is the new core component. We will create a self-contained JavaScript file that performs the actual parsing.

**Action:**
*   Create a file named `typescript-parser.js` and bundle it with the `CodeContext` application (e.g., as an "Embedded Resource" or a file copied to the output directory).

**`typescript-parser.js` Implementation:**

```javascript
// typescript-parser.js
const ts = require('typescript');

// This function reads all content from stdin.
function readStdin() {
    return new Promise((resolve) => {
        let data = '';
        process.stdin.on('data', chunk => data += chunk);
        process.stdin.on('end', () => resolve(data));
    });
}

// Main function to parse the code and transform the AST.
async function main() {
    const fileContent = await readStdin();
    const sourceFile = ts.createSourceFile(
        'virtual.ts', // A virtual filename is sufficient
        fileContent,
        ts.ScriptTarget.Latest,
        true // setParentNodes
    );

    const nodes = [];
    const edges = [];

    // The raw TS AST has circular references and is too complex.
    // We must walk it and create our own simplified graph structure.
    function visit(node) {
        // Example: Parsing a function declaration
        if (ts.isFunctionDeclaration(node)) {
            const functionName = node.name ? node.name.getText(sourceFile) : 'anonymous';
            const startLine = sourceFile.getLineAndCharacterOfPosition(node.getStart(sourceFile)).line + 1;
            const endLine = sourceFile.getLineAndCharacterOfPosition(node.getEnd()).line + 1;
            
            nodes.push({
                // id will be generated by C# host for consistency
                name: functionName,
                type: 'Function',
                filePath: 'TBD_by_host', // The C# host will inject the real path
                startLine: startLine,
                endLine: endLine,
                signature: node.getText(sourceFile).split('{')[0].trim(),
                language: 'typescript' // Crucial for multi-language support
            });
        }
        
        // Example: Parsing an import statement
        if (ts.isImportDeclaration(node)) {
            edges.push({
                // sourceId/targetId will be resolved by C# host
                type: 'IMPORTS',
                metadata: {
                    moduleSpecifier: node.moduleSpecifier.getText(sourceFile).replace(/['"`]/g, '')
                }
            });
        }
        
        // TODO: Add handlers for classes, interfaces, methods, calls, etc.
        // ts.isClassDeclaration, ts.isCallExpression, etc.

        ts.forEachChild(node, visit);
    }

    visit(sourceFile);

    // The final output MUST be a single JSON string to stdout.
    console.log(JSON.stringify({ nodes, edges }));
}

main().catch(err => {
    // Errors must go to stderr so the C# host can distinguish them from valid JSON.
    console.error(err);
    process.exit(1);
});
```

### 3. The C# `ILanguageParser` Implementation

This class acts as the bridge. It will use `System.Diagnostics.Process` to execute the script.

**Action:**
*   Follow the plan from the previous prompt to create the `CodeContext.TypeScript.Parser` project.
*   The `TypeScriptParser.cs` class will contain the process-calling logic.

**`TypeScriptParser.cs` Implementation:**
```csharp
// In project: CodeContext.TypeScript.Parser
// File: TypeScriptParser.cs

using System.Diagnostics;
using System.Text.Json; // Using System.Text.Json for AOT-friendliness

public class TypeScriptParser : ILanguageParser
{
    public string[] SupportedExtensions => new[] { ".ts", ".tsx", ".js", ".jsx" };

    // Path to the bundled node script. This should be resolved at runtime.
    private readonly string _scriptPath = "path/to/typescript-parser.js"; 

    public CodeGraph ParseFile(string filePath, string content)
    {
        var startInfo = new ProcessStartInfo("node", _scriptPath)
        {
            UseShellExecute = false,
            RedirectStandardInput = true,
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            CreateNoWindow = true
        };

        using var process = Process.Start(startInfo);
        if (process == null)
        {
            throw new InvalidOperationException("Failed to start Node.js process. Is Node.js installed and in the PATH?");
        }

        // 1. Pass file content to the script's stdin
        process.StandardInput.Write(content);
        process.StandardInput.Close(); // Signal end of input

        // 2. Read the JSON from stdout and any errors from stderr
        string jsonOutput = process.StandardOutput.ReadToEnd();
        string errorOutput = process.StandardError.ReadToEnd();

        process.WaitForExit();

        if (process.ExitCode != 0 || !string.IsNullOrEmpty(errorOutput))
        {
            // Log the error from stderr
            // Logger.LogError($"TypeScript parser failed for {filePath}: {errorOutput}");
            return new CodeGraph(); // Return an empty graph or throw
        }

        // 3. Deserialize the JSON from the script into our C# models
        var graph = JsonSerializer.Deserialize<CodeGraph>(jsonOutput);

        // 4. Post-process the graph: assign IDs and the file path
        // The script doesn't know the real file path or how to generate GUIDs
        // that are consistent with the rest of the system.
        foreach (var node in graph.Nodes)
        {
            node.id = Guid.NewGuid().ToString();
            node.filePath = filePath;
        }
        
        // TODO: Resolve edge relationships (e.g., link IMPORTS edge to file node)

        return graph;
    }
}
```

**Action:**

*   Add a project reference from the main `CodeContext` application to `CodeContext.TypeScript.Parser`.

### 2. Implement the `ILanguageParser` Interface

The core of the new project will be a class that fulfills the `ILanguageParser` contract defined in the PRD.

**Action:**
*   In the `CodeContext.TypeScript.Parser` project, create a new class: `TypeScriptParser`.

**Mapping Dictionary (TypeScript -> CodeNode):**

| TypeScript AST Node (`INode`) | `CodeNode.type` | `CodeEdge` Relationships to Capture                             |
| :-------------------------- | :-------------- | :-------------------------------------------------------------- |
| `IClassDeclaration`         | `Class`         | `INHERITS` (from heritage clauses), `IMPLEMENTS`                  |
| `IInterfaceDeclaration`     | `Interface`     | `INHERITS`                                                      |
| `IFunctionDeclaration`      | `Function`      | `CALLS` (from function body), `REFERENCES` (to types in params) |
| `IMethodDeclaration`        | `Method`        | `CALLS`, `REFERENCES`                                           |
| `IPropertyDeclaration`      | `Property`      | `REFERENCES` (to its type)                                      |
| `IArrowFunction`            | `Function`      | `CALLS`, `REFERENCES`                                           |
| `IImportDeclaration`        | N/A             | `IMPORTS` (Edge from File Node to external module/file)         |
| `ICallExpression`           | N/A             | `CALLS` (Edge from calling function/method to target)           |

### 4. Kuzu Database and Multi-Language Schema Concerns

**The Problem:** A C# class named `UserService` and a TypeScript class named `UserService` are different entities. The current `CodeNode` schema does not distinguish between them if they coincidentally share a name.

**Solution:** Add a `language` property to the `CodeNode` schema.

**Action:**
1.  **Update the `CodeNode` Model:** Add the `language` property to the C# `CodeNode` class.

    ```csharp
    public class CodeNode
    {
        // ... existing properties
        public string signature { get; set; }
        public string language { get; set; } // Add this property
    }
    ```

2.  **Update Existing C# Parser:** Modify the existing Roslyn-based C# parser to set this property.

    ```csharp
    // In CSharpParser.cs
    var classNode = new CodeNode {
        // ...
        type = "Class",
        language = "csharp" // Set the language
    };
    ```

3.  **Implement in New TypeScript Parser:** Set the language property for all nodes generated by the `TypeScriptParser`.

    ```csharp
    // In TypeScriptParser.cs
    var functionNode = new CodeNode {
        // ...
        type = "Function",
        language = "typescript" // Set the language
    };
    ```

**Database Impact (Kuzu):**
This is a non-breaking change for the Kuzu schema. Since the schema is flexible (implied by the JSON structure), adding a new property to the nodes is straightforward. This change allows for powerful, language-aware queries.

**Example Cypher Query (Post-Change):**
```cypher
// Find all test methods that call a C# method named 'GetUser'
MATCH (test_method:Method {language: 'typescript'})-[:CALLS]->(target_method:Method {name: 'GetUser', language: 'csharp'})
RETURN test_method
```
